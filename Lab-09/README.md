# Lab 09 – VPC, Subnets & Networking in AWS

## Objective
To design and configure a custom virtual network using AWS VPC and test connectivity between resources.

---

## Tasks Performed
- Created custom VPC with CIDR block
- Created public and private subnets
- Attached Internet Gateway to VPC
- Configured route tables for internet access
- Launched EC2 instances in both subnets
- Verified internet access in public instance
- Configured security groups for communication
- Created second VPC for networking
- Established VPC peering connection
- Added routes for inter-VPC communication
- Tested connectivity between instances

---

## Key Implementation
- Public subnet allowed internet access via gateway
- Private subnet restricted direct internet access
- EC2 instances deployed in different network zones
- Secure connectivity established using security groups
- VPC peering enabled communication between networks

---

## Result
Successfully implemented a complete cloud network architecture with secure communication and internet access control.

---

## Learning Outcomes
- Understanding VPC architecture
- Subnet and routing configuration
- Internet Gateway and connectivity
- Secure communication using security groups
- VPC peering concepts
