# Lab 10 – Monitoring using Amazon CloudWatch

## Objective
To monitor cloud resources using AWS CloudWatch by creating alarms, triggering them, and visualizing performance metrics.

---

## Tasks Performed
- Created a CPU utilization alarm for an EC2 instance
- Configured alarm threshold (CPU > 40%)
- Generated load on EC2 instance to trigger alarm
- Observed change from OK state to "In Alarm"
- Created a CloudWatch dashboard
- Added EC2 and RDS metrics to dashboard
- Created RDS database for monitoring

---

## Key Implementation
- CloudWatch alarm successfully created for CPU monitoring
- Load generated using stress tool to increase CPU usage 
- Alarm triggered automatically when threshold exceeded 
- Dashboard displayed real-time EC2 and database performance metrics

---

## Result
Successfully implemented monitoring and alerting system using AWS CloudWatch.

---

## Learning Outcomes
- Real-time cloud monitoring
- Alarm creation and triggering
- Performance visualization with dashboards
- Integration of EC2 and RDS metrics
