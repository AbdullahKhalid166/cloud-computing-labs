# Lab 12 – Identity and Access Management (IAM)

## Objective
To manage user permissions and roles in AWS using IAM and understand access control.

---

## Tasks Performed
- Created IAM user with ReadOnlyAccess policy
- Tested restrictions by attempting resource creation
- Verified EC2 instance creation failed due to limited permissions
- Attached AdministratorAccess policy to user
- Successfully created EC2 instance after permission upgrade
- Created IAM role for EC2 instance
- Attached IAM role to EC2 for secure access

---

## Key Implementation
- Read-only policy restricted resource creation
- Administrator policy granted full access to AWS services 
- IAM role allowed EC2 to securely access AWS services 

---

## Result
Successfully implemented user access control and role-based permissions in AWS.

---

## Learning Outcomes
- IAM users and policies
- Access control and permission levels
- Role-based access management
- Secure AWS resource interaction
